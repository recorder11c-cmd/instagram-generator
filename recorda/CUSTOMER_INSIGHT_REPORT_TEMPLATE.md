# レコルダ 顧客理解レポート雛形

作成日: 2026-07-14

この雛形は、「はじめての顧客理解レポート」33,000円メニューの納品物として使う。  
50ポイント通常アンケートの結果確認にも流用できる。

個人名・メールアドレス・LINE表示名などの個人情報は記載しない。  
企業・店舗へ渡す場合は、回答者を特定できない集計・要約だけを使う。

---

# はじめての顧客理解レポート

## 1. レポート概要

### 調査テーマ

例: レコルダのモニター制度に関する理解度と参加意向の確認

### 目的

- お客様・モニターがサービス内容を理解できているか確認する
- 参加・購入・再来店などの行動につながる障壁を見つける
- 次に改善すべき説明・導線・商品内容を整理する

### 実施概要

| 項目 | 内容 |
|---|---|
| 実施期間 | YYYY年MM月DD日〜YYYY年MM月DD日 |
| 回答数 | n件 |
| 設問数 | 5問 |
| 回答方法 | LINE経由のアンケート |
| 謝礼 | 例: 回答完了で50ポイント |
| 備考 | 回答は任意。個人を特定する形では集計しない |

---

## 2. 主要サマリー

### 今回わかったこと

1. 〇〇については、おおむね理解されている。
2. 一方で、△△については説明不足が見られる。
3. □□は参加・購入判断に影響しやすい。
4. 自由記述では、□□への不安や要望が目立つ。
5. 次の改善では、説明文・導線・FAQの見直しが有効。

### 一言まとめ

> 今回の結果から、〇〇の関心は確認できた。次は、△△の説明を補い、□□しやすい導線に整えることで、参加・購入・再来店につながりやすくなる。

## 3. 設問別結果

### Q1. 〇〇は分かりやすいですか？

| 回答 | 件数 | 割合 |
|---|---:|---:|
| 分かりやすい |  |  |
| だいたい分かる |  |  |
| 少し分かりにくい |  |  |
| 分かりにくい |  |  |

#### 読み取り

- 肯定的な回答が多い場合: 基本説明は伝わっている。
- 「少し分かりにくい」「分かりにくい」が多い場合: 最初の説明文やLP冒頭の見直しが必要。

#### 改善案

- 冒頭に「誰向けか」「何が得られるか」を1文で追加する。
- 詳細説明の前に、参加・利用の流れを3ステップで示す。

---

### Q2. 〇〇の案内が届いた場合、参加・利用したいと思いますか？

| 回答 | 件数 | 割合 |
|---|---:|---:|
| 参加・利用したい |  |  |
| 内容による |  |  |
| あまり参加・利用しないと思う |  |  |

#### 読み取り

- 「内容による」が多い場合、案内文でテーマ・所要時間・メリットを明確にする必要がある。
- 「参加したい」が少ない場合、対象者にとっての動機づけが弱い可能性がある。

#### 改善案

- 配信文に「所要時間」「得られるもの」「回答後の扱い」を明記する。
- 案内文の1行目で、対象者に関係するテーマだと分かるようにする。

---

### Q3. 交換・特典・購入条件についてどう感じますか？

| 回答 | 件数 | 割合 |
|---|---:|---:|
| 良いと思う |  |  |
| もう少し説明がほしい |  |  |
| 条件が遠い・分かりにくい |  |  |
| 興味がない |  |  |

#### 読み取り

- 条件そのものより、「いつ・どうなるか」が分からないことで不安が出ることがある。
- 特典や交換条件は、細かく説明しすぎると逆に重く見える場合がある。

#### 改善案

- FAQに「いつ付与されるか」「何に交換できるか」「現金出金の有無」を追加する。
- 配信文では短く説明し、詳細はLPやFAQへ分ける。

---

### Q4. 参加しやすい所要時間はどれですか？

| 回答 | 件数 | 割合 |
|---|---:|---:|
| 1分以内 |  |  |
| 3分以内 |  |  |
| 5分以内 |  |  |
| 内容による |  |  |

#### 読み取り

- 短時間希望が多い場合、初回は3分以内に抑える。
- 「内容による」が多い場合、テーマの関心度によって回答率が変わる。

#### 改善案

- 初回は5問以内、自由記述は任意にする。
- 配信文に「約〇分」と明記する。

---

### Q5. 自由記述の要約

#### 目立った声

- 〇〇について知りたい
- △△が分かりにくい
- □□なら参加しやすい

#### 代表的な示唆

1. 説明不足の箇所: 〇〇
2. 不安になりやすい箇所: △△
3. 改善すると反応が上がりそうな箇所: □□

---

## 4. 改善提案

### 提案1. 冒頭説明を短くする

現状の説明が長い場合、最初に読む人が離脱しやすい。  
最初の1〜2行で「誰に何を聞くのか」「回答すると何があるのか」を示す。

### 提案2. 配信文に所要時間を入れる

「短いアンケート」だけでなく、「約1〜2分」「全5問」など具体的に書く。

### 提案3. 特典・ポイントの扱いをFAQ化する

本文にすべて書くと重くなるため、本文は短く、詳細はFAQやLPに分ける。

### 提案4. 初回は小さく配信する

いきなり全員配信せず、5〜10名で反応を見てから広げる。  
問い合わせが出た場合は、案内文やLPを修正してから次の配信へ進む。

### 提案5. 次回調査テーマを分ける

今回の結果から、次回は以下のいずれかを1テーマだけ聞く。

- 参加しやすい謝礼・ポイント条件
- 回答したいテーマ
- サービス説明の分かりやすさ
- 購入・来店・利用前の不安

---

## 5. 次のアクション

| 優先度 | アクション | 目的 |
|---|---|---|
| 高 | 配信文を短く修正 | 回答率を上げる |
| 高 | FAQを1つ追加 | 不安・問い合わせを減らす |
| 中 | 5〜10名へ追加配信 | 小さく再検証する |
| 中 | 自由記述をもとに次回テーマを決める | 継続的な改善につなげる |
| 低 | LPに実績・説明を追加 | 信頼感を高める |
