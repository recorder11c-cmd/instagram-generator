# レコルダ SNS半自動化・LINE誘導・コンテンツ販売計画

作成日: 2026-07-18

## 目的

SNS投稿を毎回ゼロから考える状態をやめ、以下の流れを半自動化する。

1. レコルダの実証・学びを記録する
2. X / note / Instagram向けに分解する
3. LPへ誘導する
4. 公式LINEへ誘導する
5. 相談・モニター登録・初回レポート販売につなげる
6. 将来的にBrain等でノウハウ販売する

## 基本方針

最初から完全自動投稿にはしない。

理由:

- SNSの文脈に合わない投稿が出ると信用を落とす
- Xや各SNSの自動投稿ルール変更に影響される
- 最初は売り方・反応を見ながら調整した方がよい

初期は以下の半自動にする。

```text
実証ログ
→ 投稿案を自動生成
→ 人が確認
→ 手動投稿または予約投稿
→ LP / LINEへ誘導
→ 反応を記録
```

## 導線設計

### 1. X

役割:

- 日々の学びを短く発信する
- 「小さく聞いて改善する」考え方を広げる
- LPまたはnoteへ誘導する

誘導先:

- 基本: `https://recorder-llc.com/`
- モニター募集時: 公式LINEリンク
- 詳細説明: note記事

### 2. note

役割:

- Xより詳しく背景を書く
- 実証ログを記事化する
- 事業者に「この人は何をしているか」を伝える

誘導先:

- LP
- 公式LINE
- 個別相談

### 3. Instagram

役割:

- 図解・実績カード・事例紹介
- 地域事業者や店舗向けに見た目で伝える

初期は毎日投稿しない。

優先:

- 週1〜2枚の図解
- ストーリーでLP・LINE誘導

### 4. 公式LINE

役割:

- モニター登録
- ポイント交換
- 事業者からの問い合わせ入口

注意:

- 事業者向け相談とモニター向け案内が混ざる場合は、返信文で分岐させる
- 個別対応が増えるまでは手動確認でよい

## 投稿素材の元ネタ

毎週、以下のどれかを1つ記録する。

- 実施したアンケート
- 回答数
- 気づいたこと
- うまくいかなかったこと
- LPを直した理由
- LINE返信で詰まったこと
- ポイント制度の改善
- 営業で聞かれた質問
- 顧客理解レポートの作成過程

これを投稿素材に変換する。

## 週次運用

### 月曜: 学び投稿

例:

```text
小さなアンケートでも、説明が伝わっているかは確認できます。

大事なのは件数を大きく見せることではなく、
次に何を直すかが見えること。

レコルダでは、短いアンケートから改善提案までまとめる仕組みを作っています。
https://recorder-llc.com/
```

### 水曜: ノウハウ投稿

例:

```text
アンケートを送るときは、最初にこの3つを伝えると回答されやすくなります。

・所要時間
・回答は任意
・何に使うか

謝礼がある場合は、条件も先に伝える。
小さな透明性が安心につながります。
```

### 金曜: 事業者向け提案

例:

```text
公式LINEがなくても、顧客アンケートは始められます。

Instagram、DM、店頭QR、既存顧客への案内。

まずは5問だけ聞いて、改善点を整理する。
レコルダでは「はじめての顧客理解レポート」を準備しています。
https://recorder-llc.com/
```

### 週末: note化

Xで反応が良かった投稿を、note記事へ広げる。

## 半自動化の実装イメージ

### Phase 1: 手動＋テンプレート

今すぐ可能。

- Codexで投稿案を生成
- X / note / Instagramへ手動投稿
- 反応を手入力で記録

### Phase 2: 投稿カレンダー化

スプレッドシートまたはMarkdownで管理。

管理項目:

- 投稿日
- 媒体
- 投稿テーマ
- 投稿文
- 誘導先URL
- 投稿済み/未投稿
- 反応
- 次アクション

### Phase 3: 予約投稿

XやInstagramの予約投稿ツール、または各SNSの標準予約機能を使う。

この段階でも、投稿前の人間確認は残す。

### Phase 4: 反応分析

週1回、以下を見る。

- 表示数
- クリック
- いいね
- 返信
- LINE登録数
- 問い合わせ数

## LINE誘導の設計

SNSから直接LINEへ誘導する場合、目的を分ける。

### モニター向け

```text
短いアンケートに協力いただけるモニターを募集しています。
回答は任意で、謝礼付き調査をご案内する場合があります。

登録はこちら:
https://lin.ee/n99NOF5
```

### 事業者向け

```text
お客様の声を短いアンケートで集め、改善提案までまとめる「はじめての顧客理解レポート」を準備しています。

相談はこちら:
https://recorder-llc.com/
```

最初は事業者向けはLPへ誘導し、LINEはモニター向けを中心にする方が混乱しにくい。

## Brain販売までのロードマップ

### Step 1: 無料発信

- X 10投稿
- note 2記事
- LP公開
- 最初の個別提案

### Step 2: 実績作り

- 初回レポートを1件販売
- 納品まで完走
- 匿名事例として掲載許可を取る

### Step 3: 無料テンプレート配布

- 「5問アンケート例」
- 「配信文テンプレート」
- 「顧客理解レポート構成」

LINE登録またはLP問い合わせへ誘導する。

### Step 4: Brain販売

最初は高額にしない。

おすすめ順:

1. 980円〜2,980円のテンプレート集
2. 4,980円〜9,800円の実践マニュアル
3. 実績が増えたら19,800円前後の事業化パッケージ

## 最初に作るべき販売コンテンツ

まずは以下を無料〜低価格で作る。

```text
小さなお店のための5問アンケートテンプレート
```

構成:

1. なぜ5問でよいのか
2. 基本の5問
3. 業種別の質問例
4. 配信文テンプレート
5. 回答後の読み取り方
6. 改善提案への変換例
7. レコルダの顧客理解レポート案内

## 注意

- SNSで「稼げる」「副業で簡単」には寄せない
- Brain販売でも実績を水増ししない
- 個人情報や回答原文を出さない
- 自動投稿は必ず人間確認を挟む
- LINE誘導はモニター向けと事業者向けを混同しない
- 事業者向け相談は、最初はLP問い合わせを中心にする
