# モニター属性情報の追加方針

作成日: 2026-07-18

## 目的

モニター登録時に、任意で年代・性別・職業を取得できるようにする。  
企業向けレポートでは、個人を特定しない範囲で「どの層にどの傾向があるか」を補足する。

## 追加する項目

- 年代: 10代 / 20代 / 30代 / 40代 / 50代 / 60代以上 / 回答しない
- 性別: 女性 / 男性 / その他 / 回答しない
- 職業: 会社員 / 自営業・フリーランス / 会社役員・経営者 / パート・アルバイト / 学生 / 主婦・主夫 / 無職・休職中 / その他 / 回答しない

すべて任意。住所、勤務先、収入、詳細な家族構成など、レポートに不要な細かい属性は取得しない。

## 利用時のルール

- レポートでは匿名集計として扱う
- 個人名、メールアドレス、LINE名、回答原文と属性を結びつけて外部共有しない
- 属性別集計は、人数が少なすぎる場合は出さない
- 目安として、同じ属性の回答が5件未満の場合は「参考傾向」としても分けすぎない

## 期待できるレポート表現

- 20〜30代では「スマホで見やすいか」を重視する傾向
- 40代以上では「説明の分かりやすさ」へのコメントが多い
- 自営業・会社員で利用目的の違いが見える

ただし、初期は回答数が少ないため、断定表現ではなく「傾向」「参考」として扱う。

## 実装内容

- 登録フォームに任意項目を追加
- 登録APIで任意項目を受け取る
- Supabase `recorda_contacts` に `age_group`, `gender`, `occupation` を追加
- プライバシーポリシーに任意属性の取得と匿名集計利用を明記
